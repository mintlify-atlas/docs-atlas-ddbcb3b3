This guide covers common problems you might encounter when using marco-code and how to resolve them.

## Common Issues

<Accordion title="Error: OPENROUTER_API_KEY is not set">

**Symptom:**
```bash
RuntimeError: OPENROUTER_API_KEY is not set
```

**Cause:**
marco-code requires an OpenRouter API key to function. The key is checked when the client is first initialized:

```python title="marco_code.py:55-62"
def _get_client():
    """Return the shared OpenAI client, creating it on first use."""
    global _client
    if _client is None:
        if not API_KEY:
            raise RuntimeError("OPENROUTER_API_KEY is not set")
        _client = OpenAI(api_key=API_KEY, base_url=BASE_URL)
    return _client
```

**Solution:**

1. Create a `.env` file in your project root:
   ```bash
   touch .env
   ```

2. Add your API key:
   ```bash
   OPENROUTER_API_KEY=sk-or-v1-...
   ```

3. Verify it loads:
   ```bash
   python3 -c "from dotenv import load_dotenv; import os; load_dotenv(); print(os.getenv('OPENROUTER_API_KEY'))"
   ```

<Note>
Get an API key at [openrouter.ai/keys](https://openrouter.ai/keys)
</Note>

</Accordion>

<Accordion title="Command timed out after 30s">

**Symptom:**
```
Error: command timed out after 30s: your-long-command
```

**Cause:**
Bash commands are limited to 30 seconds by default:

```python title="marco_code.py:52"
BASH_TIMEOUT = 30  # seconds
```

The timeout is enforced in the bash tool:

```python title="marco_code.py:107-112"
def bash(command: str) -> str:
    """Execute a shell command and return stdout + stderr."""
    try:
        result = subprocess.run(
            command, shell=True, capture_output=True, text=True, timeout=BASH_TIMEOUT,
        )
    except subprocess.TimeoutExpired:
        return f"Error: command timed out after {BASH_TIMEOUT}s: {command}"
```

**Solutions:**

1. **Split the command** into smaller steps:
   ```bash
   # Instead of: "npm install && npm build && npm test"
   marco -p "run npm install"
   marco -p "run npm build"
   marco -p "run npm test"
   ```

2. **Run in background** and poll for completion:
   ```bash
   marco -p "start the build in background and check if build.log exists"
   ```

3. **Modify BASH_TIMEOUT** in your local fork:
   ```python
   BASH_TIMEOUT = 120  # 2 minutes
   ```

<Warning>
Increasing the timeout may cause marco-code to appear unresponsive during long operations.
</Warning>

</Accordion>

<Accordion title="Warning: reached maximum iterations (10)">

**Symptom:**
```
Warning: reached maximum iterations (10)
```

**Cause:**
The agent loop is capped at 10 iterations to prevent infinite loops:

```python title="marco_code.py:51"
MAX_ITERATIONS = 10
```

The loop exits after MAX_ITERATIONS even if the LLM hasn't finished:

```python title="marco_code.py:217-257"
for _iteration in range(MAX_ITERATIONS):
    chat = call_model(messages, TOOL_DEFINITIONS)
    # ... process tool calls ...
    if chat.choices[0].finish_reason == "stop":
        break
else:
    print(
        f"Warning: reached maximum iterations ({MAX_ITERATIONS})",
        file=sys.stderr,
    )
```

**Test case validation:**

```python title="test_marco_code.py:221-241"
def test_max_iterations_cap(self, mock_get_client, capsys):
    """Agent loop stops after MAX_ITERATIONS even if LLM keeps calling tools."""
    mock_client = MagicMock()
    mock_get_client.return_value = mock_client

    tool_call = _make_tool_call("bash", {"command": "echo loop"})
    looping_resp = MagicMock()
    looping_resp.choices = [_make_choice(
        content="still going",
        tool_calls=[tool_call],
        finish_reason="tool_calls",
    )]

    mock_client.chat.completions.create.return_value = looping_resp

    with patch("sys.argv", ["marco-code", "-p", "loop forever"]):
        main()

    captured = capsys.readouterr()
    assert "maximum iterations" in captured.err
    assert "still going" in captured.out
```

**Solutions:**

1. **Break down the task** into smaller prompts:
   ```bash
   marco -p "read the config file"
   marco -p "update the version field to 2.0"
   marco -p "write the config back"
   ```

2. **Be more specific** in your prompt:
   ```bash
   # Vague (may loop):
   marco -p "fix all the bugs"
   
   # Specific (completes faster):
   marco -p "fix the TypeError on line 42 of app.py"
   ```

3. **Increase MAX_ITERATIONS** in your fork:
   ```python
   MAX_ITERATIONS = 20
   ```

<Tip>
If the LLM reaches max iterations, the last message content is still printed. Check the output—it may have completed the task despite the warning.
</Tip>

</Accordion>

<Accordion title="Tool execution failed with FileNotFoundError">

**Symptom:**
```
Error executing read: [Errno 2] No such file or directory: 'myfile.txt'
```

**Cause:**
The `read` tool raises FileNotFoundError for non-existent paths:

```python title="test_marco_code.py:36-38"
def test_file_not_found(self):
    with pytest.raises(FileNotFoundError):
        read("/nonexistent/path/file.txt")
```

The exception is caught and returned as an error string:

```python title="marco_code.py:239-240"
except Exception as e:
    result = f"Error executing {fn.name}: {e}"
```

**Solutions:**

1. **Check the working directory**:
   ```bash
   marco -p "what is the current directory?"
   ```

2. **List available files**:
   ```bash
   marco -p "list all .txt files in the current directory"
   ```

3. **Use absolute paths**:
   ```bash
   marco -p "read /full/path/to/file.txt"
   ```

4. **Let the LLM self-correct**:
   The error is returned to the LLM, which can retry with a corrected path.

</Accordion>

<Accordion title="API connection issues">

**Symptoms:**
- `Connection timeout`
- `HTTP 502 Bad Gateway`
- `API rate limit exceeded`

**Solutions:**

1. **Check API status**:
   Visit [status.openrouter.ai](https://status.openrouter.ai)

2. **Verify your API key**:
   ```bash
   curl https://openrouter.ai/api/v1/auth/key \
     -H "Authorization: Bearer $OPENROUTER_API_KEY"
   ```

3. **Check your network**:
   ```bash
   curl -I https://openrouter.ai
   ```

4. **Review rate limits**:
   OpenRouter has per-key rate limits. Check your dashboard at [openrouter.ai/activity](https://openrouter.ai/activity).

<Note>
marco-code uses the `anthropic/claude-haiku-4.5` model by default. Ensure your API key has access to this model.
</Note>

</Accordion>

## Debugging Tips

### Check stderr output

Tool executions are logged to stderr with the `[tool]` prefix:

```python title="marco_code.py:233"
print(f"  [tool] {fn.name}({fn_args})", file=sys.stderr)
```

**View tool logs:**
```bash
marco -p "read config.json" 2>&1 | grep "\[tool\]"
```

Output:
```
  [tool] read({'file_path': 'config.json'})
```

### Test tools in isolation

You can import and test tools directly:

```python
from marco_code import read, write, bash

# Test read
print(read("test.txt"))

# Test write
print(write("/tmp/test.txt", "hello"))

# Test bash
print(bash("ls -la"))
```

### Run the test suite

Validate your installation:

```bash
pytest test_marco_code.py -v
```

Expected output:
```
test_marco_code.py::TestReadTool::test_reads_file_contents PASSED
test_marco_code.py::TestReadTool::test_file_not_found PASSED
test_marco_code.py::TestWriteTool::test_writes_file PASSED
...
```

### Enable verbose logging

Set the `OPENAI_LOG` environment variable:

```bash
export OPENAI_LOG=debug
marco -p "your prompt"
```

This shows full request/response details from the OpenAI client.

## Still Having Issues?

If you encounter a problem not covered here:

1. Check the [GitHub Issues](https://github.com/marco-code-ai/marco-code/issues) page
2. Review the [test suite](https://github.com/marco-code-ai/marco-code/blob/main/test_marco_code.py) for expected behavior
3. Open a new issue with:
   - Your full command
   - stderr output (with `[tool]` logs)
   - Expected vs. actual behavior

<Tip>
When reporting issues, include the output of `marco -p "show system info"` to help maintainers reproduce your environment.
</Tip>
