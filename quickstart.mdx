## Your First Command

Let's get you from zero to running your first AI-powered coding task. This guide assumes you've already [installed marco-code](/installation) and configured your API key.

<Steps>
  <Step title="Verify your setup">
    First, make sure marco-code is working:

    ```bash
    marco-code --help
    ```

    You should see the help message showing the `-p` option for prompts.
  </Step>

  <Step title="Run a simple command">
    Let's start with a basic task—creating a Python script:

    ```bash
    marco-code -p "Create a hello.py file that prints 'Hello, World!' in different languages"
    ```

    **What happens:**
    - marco-code sends your prompt to the LLM
    - The LLM decides to use the `write` tool
    - marco-code creates the file with generated content
    - The LLM responds confirming the task is complete

    **Expected output:**
    ```
      [tool] write({'file_path': 'hello.py', 'content': '...'})
    I've created hello.py with greetings in multiple languages.
    ```
  </Step>

  <Step title="Verify the result">
    Check that the file was created:

    ```bash
    cat hello.py
    ```

    Run the generated script:

    ```bash
    python hello.py
    ```
  </Step>
</Steps>

## Understanding the Output

When marco-code runs, you'll see two types of output:

**Tool calls** (stderr):
```
  [tool] write({'file_path': 'hello.py', 'content': '...'})
```
These show what tools the LLM is calling to complete your task.

**Final response** (stdout):
```
I've created hello.py with greetings in multiple languages.
```
This is the LLM's natural language response to your prompt.

<Tip>
You can redirect tool output to hide it: `marco-code -p "..." 2>/dev/null`
</Tip>

## Common Use Cases

Now that you've run your first command, try these practical examples:

### Reading and Analyzing Files

```bash
marco-code -p "Read hello.py and tell me what it does"
```

The LLM will use the `read` tool to examine the file and explain its functionality.

### Writing and Modifying Code

```bash
marco-code -p "Add a function to hello.py that takes a language code and returns the greeting"
```

The LLM will:
1. Read the current file
2. Modify it to add the new function
3. Write the updated content back

### Running Commands

```bash
marco-code -p "List all Python files in the current directory and count the lines in each"
```

The LLM will use the `bash` tool to execute shell commands and report results.

### Multi-Step Tasks

```bash
marco-code -p "Create a tests directory, write a unit test for hello.py, and run it with pytest"
```

The LLM will:
1. Create the directory using bash (`mkdir`)
2. Write a test file
3. Execute pytest
4. Report the results

<Note>
marco-code has a maximum of 10 iterations and a 30-second timeout for bash commands. This prevents infinite loops and runaway processes.
</Note>

## Complete Example Workflow

Let's build a complete mini-project:

<Steps>
  <Step title="Create a project structure">
    ```bash
    marco-code -p "Create a directory called calculator with __init__.py, calc.py with add/subtract/multiply/divide functions, and requirements.txt"
    ```
  </Step>

  <Step title="Add tests">
    ```bash
    marco-code -p "Create calculator/test_calc.py with pytest tests for all calculator functions"
    ```
  </Step>

  <Step title="Run the tests">
    ```bash
    marco-code -p "Install pytest if needed and run the tests in calculator/"
    ```
  </Step>

  <Step title="Generate documentation">
    ```bash
    marco-code -p "Read the calculator code and create a README.md with usage examples"
    ```
  </Step>
</Steps>

**Result:** You've just built a complete Python package with tests and documentation using natural language prompts!

## Environment Configuration

marco-code loads configuration from a `.env` file. Here's the template from the repository:

```bash
OPENROUTER_API_KEY=your-key-here
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
```

<Accordion title="Advanced: Using Different Models">
  marco-code uses `anthropic/claude-haiku-4.5` by default (see `marco_code.py:78`). To use a different model, you'll need to modify the source code:

  ```python
  # In marco_code.py, line 78
  response = client.chat.completions.create(
      model="anthropic/claude-haiku-4.5",  # Change this
      messages=messages,
      tools=tools,
  )
  ```

  Popular alternatives:
  - `anthropic/claude-sonnet-4.5` - More capable, higher quality
  - `openai/gpt-4-turbo` - OpenAI's latest
  - `google/gemini-pro` - Google's model

  Check [OpenRouter's model list](https://openrouter.ai/models) for all available options.
</Accordion>

## Troubleshooting

### No Response

If marco-code hangs or times out:

```bash
# Check your API key is set
echo $OPENROUTER_API_KEY

# Or check .env file
cat .env
```

### Tool Errors

If you see `[tool] ... Error: ...` messages:

- **FileNotFoundError**: The file doesn't exist—check the path
- **PermissionError**: You don't have write access—check directory permissions
- **Command timed out**: Bash command took >30s—simplify the task

### Unexpected Behavior

The LLM might misinterpret your prompt. Try:

1. **Being more explicit**: "Create a new file" instead of "add a file"
2. **Adding constraints**: "Don't delete any existing files"
3. **Showing examples**: "Create a function like this: def add(a, b): return a + b"
