## Overview

The `bash` tool executes shell commands and returns their combined stdout and stderr output.

## Function Signature

Defined in `marco_code.py:105-116`:

```python
def bash(command: str) -> str:
    """Execute a shell command and return stdout + stderr."""
    try:
        result = subprocess.run(
            command, shell=True, capture_output=True, text=True, timeout=BASH_TIMEOUT,
        )
    except subprocess.TimeoutExpired:
        return f"Error: command timed out after {BASH_TIMEOUT}s: {command}"
    output = result.stdout
    if result.stderr:
        output += "\n" + result.stderr
    return output
```

## Parameters

<ParamField path="command" type="string" required>
  The shell command to execute. Can include pipes, redirects, and other shell features.
</ParamField>

## Returns

<ResponseField name="output" type="string">
  Combined stdout and stderr from the command execution. If stderr is present, it's appended to stdout with a newline separator.
</ResponseField>

## Timeout Behavior

Commands are limited to **30 seconds** execution time (`marco_code.py:52`):

```python
BASH_TIMEOUT = 30  # seconds
```

If a command exceeds this timeout, the function catches `subprocess.TimeoutExpired` and returns an error message (`marco_code.py:111-112`):

```python
except subprocess.TimeoutExpired:
    return f"Error: command timed out after {BASH_TIMEOUT}s: {command}"
```

## Output Handling

The function combines stdout and stderr (`marco_code.py:113-116`):

```python
output = result.stdout
if result.stderr:
    output += "\n" + result.stderr
return output
```

This means:
- Normal output (stdout) comes first
- Error messages (stderr) are appended with a newline
- Both streams are returned as a single string

## Timeout Exception Handling

The agent loop also catches timeout exceptions (`marco_code.py:237-238`):

```python
except subprocess.TimeoutExpired as e:
    result = f"Error: command timed out after {BASH_TIMEOUT}s"
```

This provides a safety mechanism if the timeout exception propagates beyond the function.

## Important Notes

- **Shell=True**: Commands run through the system shell, enabling pipes, redirects, and environment variable expansion
- **Text mode**: Output is captured as text (UTF-8), not bytes
- **No exit code**: The function returns output, not the exit code. Both successful and failed commands return their output
- **Working directory**: Commands execute in the current working directory (`os.getcwd()`)

## Security Considerations

Since `shell=True` is used, be cautious with untrusted input:

```python
# Potentially dangerous if user_input is untrusted
command = f"rm -rf {user_input}"
bash(command)
```

The LLM generates commands based on its training, but always verify sensitive operations.
